# mongodb-playground ‚Äî project context for Claude

A Python + pymongo playground for experimenting with MongoDB indexes and
reading `explain()` output via MongoDB Compass. Created 2026-07-13.

It is the sample project behind a Medium article on index scans, so the
seeded dataset is the source of truth for every screenshot in that article.

## How MongoDB is running (the important part)

Compass is only a GUI ‚Äî it needs a running MongoDB **server**. The server
runs in a Docker container:

- **Container:** `mongo-playground` (image `mongo:7`), port `27017`
- **Persistent data volume:** `mongo-playground-data` (survives restarts)
- **Container engine:** **Colima** (not Docker Desktop) + the `docker` CLI,
  installed via Homebrew. Colima is a lightweight VM hosting Docker.

### Connecting Compass

Open MongoDB Compass and connect to:

```
mongodb://localhost:27017
```

The dataset lives in the **`playground`** database, **`users`** collection:
50,000 documents generated by `seed.py` with a fixed random seed, so the
counts are reproducible. Refresh Compass to see changes live.

### After a reboot

Colima does NOT auto-start. Bring the stack back with:

```bash
colima start                       # start the VM (needed after every reboot)
docker start mongo-playground      # container auto-restarts, but this is the manual fallback
```

Handy checks: `docker ps` (is mongo running?), `colima status`.

## Project layout & how to run

- `.venv/` ‚Äî Python virtualenv. **Only dependency: `pymongo`.**
- `scripts/` ‚Äî run scripts from *inside* this dir (they import `db.py`):
  - `db.py` ‚Äî shared connection helper (reads `MONGODB_URI`, `MONGODB_DB`)
  - `ping.py` ‚Äî test the connection (run this first)
  - `seed.py` ‚Äî build the 50,000-document `users` collection

```bash
cd scripts
../.venv/bin/python ping.py
../.venv/bin/python seed.py
```

The index experiments themselves are done by hand in Compass, on purpose:
the point is to watch the plan change. There is deliberately no script that
automates them.

Override connection via env vars: `MONGODB_URI`, `MONGODB_DB`.

### Careful with `seed.py`

`seed.py` **drops the `users` collection** before reseeding it. The fixed seed
means it regenerates identical data, but any indexes you created are lost. The
article's screenshots depend on this dataset, so don't run it casually ‚Äî treat
it as a reset button, not a routine step.

## Working preferences

- **Install only strictly-needed dependencies.** No convenience/nice-to-have
  packages ‚Äî justify each one, prefer the standard library, and flag optional
  deps for the user to decide rather than adding them silently.
